🧱 FASE 1 – Fundamentos de Infraestructura (Mes 1–3)

Objetivo: construir una base sólida en Linux, redes y scripting, capaz de administrar, diagnosticar y automatizar tareas básicas.

1. Linux y Hardware

Conceptos clave

Instalación y uso de Linux (Ubuntu Server o Debian) sin interfaz gráfica.

Comandos básicos: ls, cd, cp, mv, grep, find, tar, top, df, du, ss, netstat.

Administración: systemctl, journalctl, cron, ufw, iptables, sudo, chmod, chown.

Gestión de paquetes: apt, dpkg.

Procesos y logs: cómo verificar y reiniciar servicios.

Monitoreo local: htop, vmstat, iostat, free, dmesg.


Práctica

Crear 2–3 VMs con VirtualBox o Proxmox.

Configurar usuarios, permisos y firewall.

Automatizar backups con rsync o tar y un cron job.

♾️ DEVOPS: Filosofía, Rol y Ciclo

DevOps no es solo un rol técnico, sino una cultura de colaboración y automatización que une a Desarrollo (Dev) y Operaciones (Ops) para entregar software más rápido, estable y seguro.


---

👥 Roles y Diferencias

Rol	Qué hace	Enfoque	Herramientas típicas

Dev	Escribe código de aplicaciones	Funcionalidad, features	Git, VSCode, frameworks, tests
Ops	Mantiene servidores e infraestructura	Estabilidad, seguridad	Linux, AWS, Nginx, monitoreo
DevOps	Automatiza y conecta ambos mundos	Fluidez, despliegue continuo	Docker, Terraform, Kubernetes, GitHub Actions, Prometheus, Grafana


> 🧩 DevOps no crea apps, pero automatiza cómo se construyen, despliegan y mantienen vivas.




---

♾️ Ciclo DevOps Adaptado a tu Plan

Fase	Qué haces	Herramientas

PLAN 🧭	Definir features y documentar	Trello, GitHub Projects, README
CODE 💻	Escribir scripts y configuraciones	Git, Bash, jq/yq
BUILD 🧱	Construir artefactos y contenedores	Docker, Compose
TEST 🧪	Validar código automáticamente	GitHub Actions
RELEASE 🚀	Preparar despliegue automatizado	Actions + Docker Hub + Terraform
DEPLOY ☁️	Crear y desplegar infraestructura	Terraform, AWS, Kubernetes
OPERATE ⚙️	Mantener sistemas y tareas repetitivas	Kubernetes, AWS, Bash/Python
MONITOR 🔍	Observar métricas y alertas	Prometheus, Grafana, Loki


> 🔄 Los datos de MONITOR vuelven a PLAN, cerrando el ciclo de mejora continua.

💬 En Resumen

> Un DevOps no es jefe ni árbitro: es el pegamento entre Dev y Ops.
Su misión es que todo funcione, que los equipos colaboren y que el software llegue a los usuarios rápido, estable y seguro.
Convierte conflictos en datos, automatiza procesos y mantiene el ciclo ♾️ vivo, impulsando la mejora continua y el aprendizaje constante.




🎯 El Puesto Objetivo (Tu "Estrella Polar")
Tu meta principal no es un rol, sino un perfil: el Ingeniero Híbrido (Infraestructura/Cloud).
Este es el profesional que las empresas no encuentran: alguien que entiende de Redes (gracias a tu base de TUARI/CCNA) y además sabe Automatización y Nube (gracias a tu plan DevOps).
🗺️ El Camino de Roles (Paso a Paso)
Aquí tienes la "escalera" de puestos más común para llegar a ese objetivo.
Nivel 1: El Punto de Partida (Primer Empleo)
Tu primer trabajo será donde capitalizas tu base de Linux y Redes. Aquí es donde tu estudio de CCNA te da una ventaja inmediata.
 * Puestos Objetivo:
   * Analista de NOC (Network Operations Center): El rol perfecto para alguien con conocimientos de CCNA. Monitorearás y resolverás problemas de red.
   * Administrador de Sistemas (SysAdmin) Junior: El rol perfecto para tu conocimiento de Linux (EA1: Taller de Linux). Administrarás servidores.
   * Soporte Técnico Nivel 2 (Help Desk L2): Un rol donde resuelves problemas que el Nivel 1 no puede (a menudo de redes o servidores).
Nivel 2: La Transición (Segundo Empleo)
Aquí es donde haces el "salto". Usas la experiencia del Nivel 1 y le sumas el kit de herramientas moderno de tu plan (Docker, Terraform, AWS).
 * Puestos Objetivo:
   * Ingeniero de Cloud (Cloud Engineer) Junior: El nombre lo dice todo. Ayudarás a administrar la infraestructura en AWS, Azure o GCP.
   * Analista CloudOps / InfraOps: Un rol enfocado en operar la plataforma en la nube (aplicar lo de tu Fase 2 y 3).
   * NetDevOps (si vienes de NOC): Un rol más nuevo donde automatizas tareas de red con Python y Ansible.
Nivel 3: El Rol Objetivo (La Meta Profesional)
Ya tienes la base de redes/sistemas (Nivel 1) y la experiencia en nube/automatización (Nivel 2). Ahora los fusionas.
 * Puestos Objetivo:
   * Ingeniero de Infraestructura (Infrastructure Engineer): El título "híbrido" por excelencia.
   * Ingeniero de Cloud (Cloud Engineer) (Semi-Senior/Senior): El rol estándar de la industria.
   * Ingeniero DevOps: Ahora sí eres un DevOps con todas las letras, porque entiendes toda la pila, desde el cable hasta el pipeline.
Nivel 4: La Especialización (El Siguiente Nivel)
Una vez que dominas el Nivel 3, puedes especializarte en roles aún más cotizados.
 * Puestos Objetivo:
   * SRE (Site Reliability Engineer): Te enfocas en la confiabilidad, escalabilidad y rendimiento de las aplicaciones (como un DevOps, pero con un foco extremo en métricas y resiliencia).
   * Platform Engineer: Construyes las "plataformas" internas (como un Kubernetes personalizado) para que los desarrolladores de tu empresa desplieguen su código fácilmente.
   * Arquitecto de Soluciones Cloud (Cloud Solutions Architect): Te alejas un poco de la ejecución y te centras en el diseño de las soluciones en la nube.
